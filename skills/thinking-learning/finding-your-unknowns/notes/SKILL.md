# Notes(实施中 · 偏离日志)

`plan`(⑧)出了计划,实施中未知仍埋在领土里、会冒出来。本技能维护一份**运行日志**,撞上 edge case、被迫偏离计划时记一笔——不停,记完继续干。这份日志后续被 `wrapup`(⑩ 推介文档)直接引用,替评审省一遍挖未知的过程。

## "最保守"判据(核心)

撞上冲突、被迫选时,**按序取舍选最保守的那个**:

1. **不引入新依赖 / 新抽象**
2. **不改变对外契约**(API / 数据格式 / UI 行为)
3. **可回退**(feature flag / 配置项 / 环境变量)
4. **行为最接近原计划**

四条都没法满足 → **停下来问用户**,别自己拍。

## 确认门

不停——记完继续往下干,不等用户。只有"四条判据都没法满足"时才停下问。

## ⑨ 实施笔记 Implementation notes
**何时用**:实施过程中撞上 edge case,被迫偏离计划。
**产出**:`implementation-notes.md`(相对**项目根 / 用户 CWD**),含 `## Deviations` 段,每条:
- 计划是什么
- 实际遇到什么
- 选了什么(按上面判据选最保守)
- 为什么
**Prompt 模板**:
```
在整个实施过程中,维护一个 implementation-notes.md 文件。
如果撞上 edge case、被迫偏离计划:
1. 选最保守的那个方案
2. 在「Deviations」部分记一笔:计划是什么、实际遇到什么、选了什么、为什么
3. 继续往下干,别停下来等我
```
**示例条目**:
```
### 用户头像缓存
- 计划:Redis,TTL 1h
- 实际:项目无 Redis 依赖,引入成本高
- 选择:先用内存缓存(lru-cache),TTL 不变
- 原因:保守,不引入新依赖,后续可平滑切 Redis
```

---

## 衔接(跑完一条后)

- 记完继续实施,**不停下问**(除非四条判据都没法满足)
- 实施完成 → 进 `wrapup`:⑩ 推介文档会**直接引用** `implementation-notes.md` 里的关键决策,预答评审会担心的失败点
- 若偏离触及**不可逆 / 高爆炸半径**决策(如改对外契约)→ 这是未知未知信号,停下问用户

## 跑完必做

- 产出匹配用户语言(中文则中文产出)
- 每条偏离必须写全四项(计划 / 实际 / 选择 / 原因),缺一不可
- 选择必须符合"最保守"判据,或明确标注"已停下问用户"
- 记完**继续干,别停**(除非判据耗尽)
