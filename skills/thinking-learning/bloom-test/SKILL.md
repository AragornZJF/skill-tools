# 🧪 布鲁姆认知六层次测试

> 一题一答，逐层递进。用户答完一层后 AI 即时反馈，再进入下一层。**不跳层、不一次性全抛。**

---

## 设计哲学

- **一题一答，逐层递进**：每层只出一题，用户答完即时反馈，再进入下一层。
- **掌握前一阶，才能进下一阶**：记忆为基础，创造为顶峰。用户若在低层就卡住，高层问题会难以作答——这是自然的诊断信号。
- **布鲁姆六层次**：记忆 → 理解 → 应用 → 分析 → 评价 → 创造。每层要求的认知复杂度递增。

---

## 选择题模式

本 skill 默认在 **记忆层** 与 **理解层** 使用终端单选题，让用户通过输入 `A/B/C/D` 快速作答；**应用层及以上** 仍使用开放题，以检验解决复杂问题的能力。

**选择题设计规范：**

- 每题 4 个选项，仅 1 个正确答案
- 干扰项必须反映**常见误解**，具备诊断价值，而非随机错误
- 用户输入字母即可，AI 即时判定对错并给出解析
- 选错后给出正确答案与逐项解析，按「错误/空白，给出答案」记 **40 分**，不卡关，继续下一层

**为什么前两层用选择题：**

- 记忆层适合快速检测事实识别
- 理解层可通过「选择最佳比喻/解释」检测用户是否真正 grasp 核心机制
- 应用层及以上需要步骤、推理、设计，开放题更有效

| --------------- | ------------------------ | ------------------------------------------------------------------------ |

## 六层次出题流程

每层 AI 输出固定格式，逐层递进。用户作答后立即反馈，确认后再出下一题。

### 🧩 第一层 · 记忆（Remembering）

**目的**：检查用户是否能准确回忆或识别核心事实、术语、定义。

**问题设计原则**：

- 使用单选题，4 个选项，仅 1 个正确答案
- 围绕定义、关键文件、核心特征出题
- 干扰项应反映常见误解，帮助诊断薄弱点
- ✅ "以下哪个选项最准确地描述了 [概念] 的定义？"
- ✅ "[概念] 的三个核心特征包括以下哪一组？"
- ❌ "你能说说 [概念] 大概是什么吗"（太模糊）

**通过标准**：选择正确答案，能识别核心事实与术语。

**输出格式**：

```
### 🧩 第一层 · 记忆（Remembering）
请选择一个最准确的答案：

**问题**：[关于定义、关键文件或核心特征的问题]

A. [正确选项]
B. [常见误解干扰项]
C. [部分正确但遗漏关键点的干扰项]
D. [表面相关但错误的干扰项]

请输入 A/B/C/D：
```

---

### 🔍 第二层 · 理解（Understanding）

**目的**：检查用户是否真正理解概念的含义，而不仅仅是背诵。

**问题设计原则**：

- 使用单选题，4 个选项，仅 1 个正确答案
- 通过「选择最佳比喻/解释/推断」来检测理解深度
- 保留比喻能力检测：题目可要求从多个比喻中选出最贴切的一个
- 干扰项应反映「只背名字不理解机制」的典型错误
- ✅ "以下哪个比喻最能说明 [概念] 的核心机制？"
- ✅ "关于 [概念]，以下哪种解释最准确？"
- ❌ "你理解 [概念] 吗"（只能答"是/否"，无诊断价值）

**通过标准**：选择最能抓住核心机制的答案，而非表面描述。

**输出格式**：

```
### 🔍 第二层 · 理解（Understanding）
请选择一个最准确的答案：

**问题**：以下哪个比喻/解释最能说明 [概念] 的核心机制？

A. [正确比喻/解释]
B. [只描述表面现象的干扰项]
C. [混淆关键机制的干扰项]
D. [过度简化或错误类比的干扰项]

请输入 A/B/C/D：
```

---

### 🛠️ 第三层 · 应用（Applying）

**目的**：检查用户能否在具体场景中使用该概念解决问题。

**问题设计原则**：

- 给出一个（或让用户提供）具体场景
- 要求用户说明如何用该概念解决实际问题
- ✅ "在 [应用场景] 使用 [概念] 解决问题，请描述你的步骤"
- ✅ "举一个你工作中可以用 [概念] 的例子，说明怎么用"
- ❌ "[概念] 有什么应用"（变成列举，不是解决问题）

**通过标准**：步骤合理、场景匹配、能说明具体操作而非泛泛而谈。

**输出格式**：

```
### 🛠️ 第三层 · 应用（Applying）
**问题**：在 [应用场景] 使用 [概念] 解决问题，请描述你的步骤。
```

---

### 🔬 第四层 · 分析（Analyzing）

**目的**：检查用户能否分解概念的结构，理清各部分之间的关系及其对整体的影响。

**问题设计原则**：

- 要求分解、归因、关联
- 用 [应用场景] 作为分析对象，降低抽象度
- ✅ "将 [知识点] 分解成零件，确定零件的关联关系及对整体结果的影响。以 [应用场景] 为例说明。"
- ✅ "分析 [某案例] 中 [概念] 的各个部分如何相互关联"
- ❌ "分析一下 [概念]"（太宽泛）

**通过标准**：分解合理、能说清因果关系、能指出部分如何影响整体。

**输出格式**：

```
### 🔬 第四层 · 分析（Analyzing）
**问题**：将 [知识点] 分解成零件，确定零件的关联关系及对整体结果的影响。以 [应用场景] 为例，说明各部分如何相互关联、如何影响整体。
```

---

### ⚖️ 第五层 · 评价（Evaluating）

**目的**：检查用户能否基于证据做出判断、批判或权衡。

**问题设计原则**：

- 给出一个观点/论断（或由用户指定）
- 要求评估证据权重、做出批判性判断
- ✅ "对 [某观点] 给出证据权重评估与批判"
- ✅ "[某争议] 中支持和反对 [概念] 的论据各有什么力度？"
- ❌ "你觉得 [某观点] 对吗"（只问立场，不要求论证）

**通过标准**：不是简单站队，而是展示证据对比、权重分析、逻辑论证。

**输出格式**：

```
### ⚖️ 第五层 · 评价（Evaluating）
**问题**：对 [某观点] 给出证据权重评估与批判。
```

---

### 🚀 第六层 · 创造（Creating）

**目的**：检查用户能否融合多个领域知识，设计创新方案。

**问题设计原则**：

- 要求综合、设计、创新
- 融合跨学科原理，避免"换个场景再应用"
- ✅ "融合 [跨学科原理] 设计一个创新方案"
- ✅ "结合 [学科A] 和 [学科B] 的方法，设计一个基于 [概念] 的新系统"
- ❌ "用 [概念] 设计一个方案"（没有跨学科融合要求）

**通过标准**：展示跨学科整合能力、方案有创新性、逻辑自洽。

**输出格式**：

```
### 🚀 第六层 · 创造（Creating）
**问题**：融合 [跨学科原理] 设计一个创新方案。
```

## 反馈规则

### 反馈三要素

每条反馈包括：

1. **状态标记**：✅ 通过 / ⚠️ 部分正确 / ❌ 未掌握
2. **具体分析**：指出哪里对、哪里错
3. **后续指引**：通过 → 直接进入下一层（不问「是否进入」）；部分正确 → 追问引导；未掌握 → 给出答案+建议复习+直接进入下一层

### 选择题反馈规范

记忆层与理解层为单选题，反馈按以下规则处理：

- **选对**：状态记为 ✅ 通过，得分 **100**，简要解析正确原因，**直接出下一层题（不问「是否进入下一层」）**
- **选错**：状态记为 ❌ 未掌握，得分 **40**，**重点解析用户所选的那个干扰项为什么错**（2-3 句），其他干扰项一句话带过，避免信息过载；不卡关，直接出下一层题
- 单选题不涉及「部分正确」状态，因此该层不触发追问循环

**示例：**

```
### 反馈
✅ **通过**。A 正确——Ralph 通过文件系统持久化状态，上下文清除后仍能从文件恢复计划。
（自动出下一层题）
```

```
### 反馈
❌ **未掌握**。正确答案是 **A**。
你选了 **B**——这其实把 Ralph 当作普通单次对话工具，忽略了「循环 + 文件状态」这两个核心机制，是最常见的误解。
（C 只提到背压，是质量保障而非状态保持；D 误以为上下文清除会导致失忆，但记忆通过文件持久化。）
建议复习「文件即状态」与「上下文智能管理」部分。我们继续下一层。
```

### 各状态撰写规范

| 状态        | 语气      | 长度   | 示例                                                                                   |
| ----------- | --------- | ------ | -------------------------------------------------------------------------------------- |
| ✅ 通过     | 肯定+简明 | 1-2 句 | "✅ **通过**。定义准确，关键特征都覆盖到了。（自动出下一层题）"                        |
| ⚠️ 部分正确 | 鼓励+追问 | 2-3 句 | "⚠️ **部分正确**。定义的核心部分是对的，但遗漏了 [X]。你能说说 [X] 和 [Y] 的区别吗？"  |
| ❌ 未掌握   | 直接+友善 | 2-3 句 | "❌ **未掌握**。参考定义：[简明答案]。建议先复习 [相关概念]，我们继续下一层，没关系。" |

### 追问技巧

- ⚠️ 时不直接给答案，先给 1-2 个**引导问题**让用户自己补充
- 用户两次补充仍不完整，再给出参考答案
- 修正后该层得分记为 **70**（而非 100 或 60）

---

## 评分细则

| 条件                 | 单层得分 |
| -------------------- | :------: |
| 首次回答准确完整     |   100    |
| 部分正确，追问后修正 |    70    |
| 部分正确，未修正     |    60    |
| 错误/空白，给出答案  |    40    |

**总分 = 6 层得分之和 / 6**，四舍五入取整。

**状态对应**：

- 总分 ≥ 80 → ✅ 掌握良好
- 60 ≤ 总分 < 80 → ⚠️ 基本掌握，有薄弱点
- 总分 < 60 → ❌ 需系统复习

## 注意

- **不卡关**：即使用户某一层未掌握，给出答案后仍进入下一层，不中断流程。
- **中文输出**：所有问题、反馈、汇总均用中文，除非用户另有要求。
- **每层确认**：反馈后**直接出下一题**，不要问「是否进入下一层」，保持流程紧凑。
- **placeholder 自动填充**：用户未指定的 placeholder 按上表默认策略自动补全，并在出题时告知用户（如"以电商场景为例"）。
