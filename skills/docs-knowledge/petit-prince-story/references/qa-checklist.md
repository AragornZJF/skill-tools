# QA Checklist

## 必过项
- 默认 600×800 竖卡（或用户指定的 size）。
- 描边 `#141414`（line）/ 调色板深色（watercolor）。
- line 模式每幕**恰好一个**朱砂红焦点，色号严格 `#C0392B`（画龙点睛，不多不少；金句/标题保持黑）。
- 复用统一定义的小王子，每幕一致。
- 迎客角色按决策表正确映射。
- 金句 ≤12 字、纯中文。
- 寓意 ≤12 字、纯中文，位于金句下方（`font-size="12"`，`fill="#666666"`）。
- 星点 ≤12。
- 辅助惊喜元素每幕 ≤1 个 + 氛围层每幕 ≤1 个（互不冲突，不超额）。
- 辅助惊喜元素使用 `svg-style.md` 定义片段，不自行创作未定义道具。
- 可选道具（路 / 脚印 / 火山 / 望远镜 / 月亮）仅在意境匹配时使用，不硬塞。
- 每位迎客角色在全篇最多出现一次，无重复客人。
- 相同客人多幕时，星空 / 小行星 / 指路星 / 氛围层布局均不同，无两张图视觉雷同。
- 启用 V1–V6 任意一种时，分镜表有对应语义理由。
- 跨幕不使用重复的惊喜变体 / 氛围层 / 布局编号。
- 文字与图形、图形与图形无重叠 / 不出框。
- 幕数 ≤8（超出已合并次要章节）。
- 纯 `<svg>`，以 `<svg` 开头，带 viewBox，不套 HTML。

## 失败信号（出现就重生成）
- 默认画成横图。
- 描边 `#1a1a1a` 或随意取色。
- line 模式一幕出现 ≥2 处朱砂红 / 把金句或标题涂红 / 红号不是 `#C0392B`。
- 每幕小王子长得不一样。
- 角色映射错（如「爱」画了狐狸、「洞察」画了玫瑰）。
- 金句超字 / 带法语 / 写成整句。
- 加了辅助惊喜元素 / 可选道具列表之外的道具（心心 / 对话气泡 / 法文 / 枯木 / 苹果树 / 标题角标等）。
- 相同客人幕之间仅文字不同，环境完全一致。
- 连续两幕用了相同的惊喜变体 / 氛围层 / 布局。
- 氛围层超出 1 个 / 辅助惊喜超出 1 个。
- 启用 V1–V6 布局但无语义理由。
- 在不该出现的场景用了可选道具（如点灯人幕加火山）。
- 元素重叠或出框。
- 套了 HTML / 没用 viewBox。

## 迭代方法
- **角色错**：回 `characters.md` 决策表重选。
- **太挤**：删星点 / 小行星 / 装饰，宁缺毋滥。
- **重叠**：拉开王子与客人间距（≥80px），文字带下移。
- **金句超字**：再提炼到 ≤12 字；抽不出就用章节标题。
- **小王子不一致**：直接复用 `characters.md` 的 `#prince` 片段，不要手绘新版。
- **惊喜变体重复**：在同一部作品中换用同一元素的不同变体。
- **氛围重复**：换一种氛围层（星环↔光晕↔薄雾↔影子）。
- **布局雷同**：如果标准布局导致跨幕雷同，考虑启用 V1–V6 并注明理由。
- **构图无惊喜**：先调惊喜元素，不够再加氛围层，还不够才动布局——逐层增强，不要一开始就推倒重来。

## 交付判断
好分镜：一眼看出「这章讲什么、谁在场」，安静不拥挤。
不合格：像花哨插画，或元素打架、看不清主次。
