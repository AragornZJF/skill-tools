# 场景配方与合成

## 竖卡解剖（600×800，坐标相对 viewBox）

```
┌─────────────────────┐ y=0
│  ✦   星空带 (上55%)  │   稀疏手绘星(点+十字) + 尘 + ≤2 小行星；宁缺毋滥
│      *    ✦         │
├─────────────────────┤
│   🤴      🦊         │   沙地弧(下25%)：王子左中(x≈235)、客人右(x≈340–470)
│  ─────星球表面────── │   两者间距 ≥80px
│   章节标题          │   文字带(下15%)：标题字 + 细分隔线 + ≤12字金句
│   ─── ───           │
│   用心去看·才看得清楚│
└─────────────────────┘ y=H   内框 inset ~16
```

- **星空带**：星点稀疏（点 + 十字两种，总数 ≤12），透明度分层；忌密。
- **沙地弧**：一条柔和曲线作星球表面；王子立于左中，客人立于右侧，间距 ≥80px。
- **文字带**：标题（标题体）+ 1.5px 细分隔线 + 金句（衬线，字间距拉开）。
- **内框**：距边 ~16 的细描边，低透明度。

> 完整骨架见 `scene-template.svg`（含统一定义的小王子 + 狐狸示例客人 + 注释）。

## 自适应尺寸（`size` 参数）
所有坐标以 viewBox 表达、按比例放置。用户改 size **只改 viewBox 维度**（与顶层背景 rect / 内框尺寸同步），**不重算角色坐标**——构图由 SVG 的 viewBox 缩放机制等比自适应。
- 极端横宽比时仍保证：文字带在底部、星与角色不出框。
- 长卷则高 = 幕数 × 卡高。

## 合成（`composite` 参数：长卷 / 网格）

### 长卷 `composite=scroll`
各幕竖向堆叠：等宽（= 卡宽），高 = 幕数 × 卡高；幕与幕之间夹一道**窄星带分隔**（3-4 个点/十字即可）。阅读顺序 = markdown 章节顺序。存 `scroll.svg`。

### 网格（九宫格）`composite=grid`
**每行固定 3 幕，放不下就换到下一行**（行数 = ⌈幕数/3⌉；6 幕即 3 列 × 2 行，9 幕即 3×3）。每格 = 一张竖卡（600×800）；整体 viewBox 宽 = 3×600、高 = ⌈幕数/3⌉×800；格间加淡分隔线、各格自带内框。阅读顺序：从左到右、从上到下。文件名按行数命名（如 `grid-3x2.svg`）。

### 网格生成的坑（重要）

1. **不要用 `<image href="...">` 引用单张 SVG** — 在 `file://` 协议下浏览器会拦截跨文件 SVG 加载（CORS 限制），网格全白。**必须内联**：读取每个卡 SVG，剥离 `<svg...>` 和 `</svg>` 标签，将中间内容插入 `<g transform="translate(x, y)">`。

2. **内联时注意 viewBox 偏移** — 每个 `<g>` 的 translate(x, y)  = (列号 × 卡宽, 行号 × 卡高)。卡内原有坐标（如 `x=300` 相对卡自身）无需改动。

3. **分隔线不要与内框重叠** — 内框在各卡最外层（`x=16, y=16, w=568, h=768`），分隔线应跨整个网格行/列（如 `<line x1="0" y1="800" x2="1800" y2="800"/>`），线宽 0.5-1px、透明度 ≤0.15，以免与内框形成双线视觉干扰。

4. **最后一行不足 3 幕时留空** — 不要拉伸、不要补占位图。右侧空区域背景色应与网格背景一致。

5. **网格背景色 vs 卡背景色** — 网格背景用 `#f5f0e8`（纸色），每卡内已有 `rect fill="#fbfaf6"`（卡色），两者不同才有层次。

6. **内联后文件体积** — 5 幕内联约 20KB，可接受。超过 9 幕时考虑用 `<use>` + `<defs>` 共享重复元素（小王子、沙地弧）来减重。

> 两种合成都用 `<defs>` 定义统一的星空 / 沙地 / 小王子 / 客人，各幕 `<use>` 引用，避免重复；最后一行不足 3 幕时右侧留空即可，不要拉伸。
