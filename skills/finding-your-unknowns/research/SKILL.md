# Research(研究域 · 替用户答)

解决 **已知未知(能问不能答)**——用户知道自己没想清楚、能问。agent 角色 = **研究伙伴**,替用户把答案挖出来。

> 本域按信号(能问不能答)拉起。进来后按下面四招跑。

## 通用第 0 步(①⑤⑥ 必须做)

①⑤⑥ 这几招都会"凭感觉编"。跑之前**必须先用 `Glob`/`Grep`/`Read` 扫描目标模块的真实代码**,每条发现都挂到具体文件/函数/配置,不得凭通用知识编造。下面每招不再重复这条。

## 确认门

内联对话类——**直接跑**,跑完允许纠偏。不先问用户"要不要跑"。

---

## ① 盲区扫描 Blindspot pass
**何时用**:进陌生代码库 / 干不熟的活 / 心里没底。
**产出**:内联对话——若干张盲区卡片 + 一句拼好的、可复制的改进 prompt。
**Prompt 模板**:
```
我正在[做 X 任务],但我对[Y 模块/领域]一无所知。
帮我做一次 blindspot pass:
1. 先用 Glob/Grep/Read 扫描 Y 的真实代码
2. 找出我可能存在的 unknown unknowns,每条挂到具体文件/函数
3. 逐条解释(是什么、为什么是坑、我该怎么问)
4. 基于这些盲区,把我的 prompt 改写成更好的版本
```
**关键**:用原词 `blindspot pass` 和 `unknown unknowns`;交代清你是谁、你知道什么,它才能对症下药。

## ② 教我未知 Teach me my unknowns
**何时用**:连"问什么"都不知道(如不懂调色却要调色)。先学词汇再谈选择。
**产出**:MD 讲解——词汇阶梯(外行词→行话)+ 成对示例(好/差对照)+ 一份"现在你能问的精准问题"清单。
**Prompt 模板**:
```
我不知道什么是[领域],但我需要[做 Z]。
帮我理解我在[领域]方面有哪些 unknown unknowns,
搭一组词汇阶梯让我能看懂行话,
再给我一份"现在我能向你提的精准问题"清单。
```

## ⑤ 头脑风暴 Brainstorm the intervention
**何时用**:知道有问题(如用户流失),但不知道该上哪个干预。
**产出**:内联对话——基于代码库的若干干预方案,按"今天就能上 → 季度级投入"排布,每条带"共鸣"勾选框,勾选的自动拼成回复。
**Prompt 模板**:
```
围绕[问题:如用户流失],先 Grep 我们的代码找相关入口,
再给我约 10 个根植于这些代码的干预方案,
按投入从"今天就能上"到"季度级投入"排布,
每条说清:改什么(挂文件)、影响面、预期收益。
我用"共鸣"勾选框挑出喜欢的,你帮我拼成回复。
```

## ⑥ 反向采访 The interview
**何时用**:需求模糊、有歧义,需要被问而不是被答。
**产出**:内联对话——**一次只问一个问题**,按"答案会改变架构"的程度排序;问完给一张决策表 + 可粘贴的实施 prompt。
**Prompt 模板**:
```
一次只问我一个问题,专挑那些模糊的、有歧义的地方。
优先问"我的回答会改变架构设计"的问题,
纯粹是细节的往后放。
问完汇总成决策表,并给我一份可粘贴的实施 prompt。
```
**关键**:给足上下文(问题是什么、想到哪步、约束是什么),它问得才准。

---

## 衔接(跑完一招后主动提议)

- **陌生 + 模糊的学习阶梯**:同一用户在同一陌生领域,往往按 **① 扫盲区 → ② 学词汇 → ⑥ 反向采访** 顺序串行跑
- ①② 跑完仍模糊,或 ⑤ 列完挑不下 → 进 **⑥ 反向采访**(本域的澄清 hub)
- 研究域跑完,未知基本挖清 → 汇到 `plan`(⑧ 可调计划)进实施
- 若跑完发现"说不清要什么、看到才知道" → 这是未知已知(看到才知道),交 `directions` 处理

## 跑完必做

- 产出匹配用户语言(中文则中文产出)
- 结构化输出(盲区卡片、决策表)
- 内联对话技法末尾给一句"可复制的更好 prompt"
- **主动提议下一步**,形成闭环
